# question_tree.json Contract

**Artifact:** `question_tree.json`
**Location:** `.research/run-NNN-TIMESTAMP/scope/question_tree.json` (per D-03/D-05)
**Format:** JSON
**Producer:** research-orchestrator (scope planning phase)
**Consumer(s):** research-collect (source discovery), research-synthesize (section ordering, INV-03)
**Validated by:** `validate_artifact.py` against `question_tree.schema.json`

> As of Phase 10, this artifact lives under `scope/` in the run directory.

## Purpose

Layered, recursive representation of the research questions produced during scope
planning. Encodes the 7-layer investigation taxonomy (identity, purpose, mechanics,
relations, comparison, evidence, open questions) and the cross-entity bridge
questions generated from graph centrality or subtopic fallback (REL-09).

The tree is the single source of truth for:

- **Collection priorities** — branches with higher `branch_priority` receive more crawl budget.
- **Synthesis section order** — the synthesizer walks `layers_populated` in taxonomy order (D-12)
  and orders intra-layer concepts by centrality (D-13).
- **Gate 1 invariant** — the orchestrator auto-regenerates (up to 2 attempts per D-19) if
  `layers_populated` contains fewer than 3 layers (D-16).

## Root Object Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| root | node | Yes | The L0 root node — its `question` is the research topic; `depth=0`. |
| layers_populated | array of strings (enum) | Yes | Layers with at least one L1 child. minItems 3 (D-16). |
| generation | object | No | Provenance for bridge-question generation. |
| generation.method | enum | No | `graph_centrality` (D-10) or `subtopic_fallback` (D-11). |
| generation.top_n | integer ≥1 | No | Count of entities used (hard ceiling 10 per Pitfall 2). |
| generation.regenerate_attempts | integer ≥0 | No | Number of Gate 1 auto-regen attempts consumed (D-19; cap 2). |

## Node Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| question | string (non-empty) | Yes | The question text. |
| layer | enum | Yes | One of the 7 investigation layers. |
| depth | integer 0-3 | Yes | Must match tree position: L0=0, L1=1, L2=2, L3=3. |
| expected_source_types | array | No | Subset of `docs`, `paper`, `blog`, `code`, `forum`, `other`. |
| branch_priority | integer ≥1 | No | 1 = highest priority. |
| children | array of node | No | Recursive child nodes. |

`additionalProperties: false` applies to both the root object and every node — unknown
fields cause validation failure (T-10-01 mitigation).

## The 7 Layers

| # | Layer | Typical questions |
|---|-------|-------------------|
| 1 | identity | What is X? Who uses X? When did it emerge? |
| 2 | purpose | Why does X exist? What problem does it solve? |
| 3 | mechanics | How does X work? What are its components? |
| 4 | relations | How does X relate to Y? (Bridge questions live here — REL-09) |
| 5 | comparison | How does X compare to Y? What are the trade-offs? |
| 6 | evidence | What evidence supports claims about X? Benchmarks, case studies. |
| 7 | open questions | What is unknown or contested about X? |

## Bridge Questions (REL-09)

Bridge questions are L1 nodes under the `relations` layer. They are generated by
pairing the top-N entities (default 5 → C(5,2) = 10 pairs) from either graph
centrality (D-10, after collection) or the `plan.json.subtopics[]` priority list
(D-11, first run before any graph exists).

All bridge questions use the canonical phrasing:

> What is the relationship between {A} and {B}?

The `top_n` parameter is clamped to ≤10 (Pitfall 2; threat T-10-03 mitigation).

## Example

```json
{
  "root": {
    "question": "Distributed consensus algorithms for edge computing",
    "layer": "identity",
    "depth": 0,
    "branch_priority": 1,
    "children": [
      {
        "question": "What is Raft?",
        "layer": "identity",
        "depth": 1,
        "expected_source_types": ["docs", "paper"],
        "branch_priority": 1
      },
      {
        "question": "Why was Raft designed as an alternative to Paxos?",
        "layer": "purpose",
        "depth": 1,
        "expected_source_types": ["paper", "blog"],
        "branch_priority": 2
      },
      {
        "question": "How does Raft leader election work under partition?",
        "layer": "mechanics",
        "depth": 1,
        "branch_priority": 1,
        "children": [
          {
            "question": "What triggers an election timeout?",
            "layer": "mechanics",
            "depth": 2,
            "expected_source_types": ["docs", "code"],
            "branch_priority": 3
          }
        ]
      },
      {
        "question": "What is the relationship between Raft and Paxos?",
        "layer": "relations",
        "depth": 1,
        "expected_source_types": ["paper"],
        "branch_priority": 2
      },
      {
        "question": "What is the relationship between Raft and Byzantine fault tolerance?",
        "layer": "relations",
        "depth": 1,
        "expected_source_types": ["paper"],
        "branch_priority": 3
      },
      {
        "question": "How do etcd and CockroachDB compare in their use of Raft?",
        "layer": "comparison",
        "depth": 1,
        "expected_source_types": ["docs", "blog"],
        "branch_priority": 2
      }
    ]
  },
  "layers_populated": ["identity", "purpose", "mechanics", "relations", "comparison"],
  "generation": {
    "method": "subtopic_fallback",
    "top_n": 5,
    "regenerate_attempts": 0
  }
}
```

## Validation

- Validator: `validate_artifact.py` against `question_tree.schema.json`.
- Timing: After orchestrator builds the tree; before Gate 1 checkpoint.
- Failure handling: Up to 2 auto-regenerate attempts (D-19); after that, downgrade
  to `warn` status and proceed (logged to `logs/run_log.md`).
